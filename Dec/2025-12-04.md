# DevOps
    - 개발 + 운영을 자동화하고 빠르게 배포하는 협업 문화

# EC2 : AWS가 제공하는 원격 컴퓨터(여기에 Docker 설치) 
    -> Docker : EC2 위에 Docker 컨테이너를 띄움

# Docker
    - 어디서나 똑같이 실행되는 "프로그램 실행 상자" 기술
    - Dockerfile : 도커 환경을 만드는 설계도(레시피)
    - Docker Image : 설계도로 만든 완성된 실행 환경(틀)
    - Docker Container : 이미지를 실제로 실행한 프로그램(실행 중인 인스턴스)
    - Docker Daemon : 도커 명령어를 뒤에서 실제로 처리하는 엔진(자동 셰프)
    - overlayFS : 여러 폴더를 덮어씌워서 하나처럼 보이게 만드는 파일시스템(이미지 + 컨테이너 변경 내용을 하나의 파일시스템처럼 쓸 수 있음)

# Docker Compose
    - 여러 개의 Docker 컨테이너를 한 번에 만들고 실행,관리하는 도구
    - 여러 컨테이너를 세트로 묶어주는 자동 실행 스크립트
    - docker compose up : 모든 컨테이너를 자동으로 생성 + 실행, 서비스 전체를 한 번에 켬

# port : 한 컴퓨터 안에서 여러 서비스가 섞이지 않게, 각각에게 부여하는 번호(어떤 프로그램에 접속할 지 포트 번호로 판단)

# RDS : AWS에서 제공하는 클라우드 데이터베이스 서버, AWS에서 빌린 DB
    - 내가 직접 DB 서버를 설치, 관리하지 않아도, AWS가 대신 설치,백업,운영 해주는 서비스

# 고결함성과 내결함성
    - 고가용성 : 서비스가 절대 안꺼지게
    - 내결함성 : 뭔가 고장나도 전체는 멀쩡하게 돌아가게

# S3 : Simple Storage Service
    - 인터넷에 있는 무한 용량의 파일 저장소
    - 내 컴퓨터가 아닌 AWS 클라우드에 있는 큰 폴더에 안전하게 저장해두는 서비스
    - bucket : 저장 공간의 이름
    - key : 파일 경로
    - s3 url 접근 : HTTP url 형태로 접근 가능(https://bucket-name.s3.amazonaws.com/key) = 파일을 인터넷에서 가져오는 주소
    - 중요한 데이터는 서버(EC2)에 저장하지 않고, RDS(데이터 베이스)와 S3(파일 저장소)에 따로 저장한다.

# RDS와 S3에 저장하는 데이터 차이
     - RDS : 표 형태의 구조적 데이터(데이터 베이스)
     - S3 : 파일 형태의 비구조적 데이터(보관함)

# IAM(Identity and Access Management) : 누가 무엇을 할 수 있는지 정하는 보안 시스템(사람/서비스에게 권한을 주고, 접근을 통제하는 관리자)
    - IAM User : 사람, 프로그램, AWS 콘솔에 로그인할 때 쓰는 아이디, githubaction에 부여할 아이디,,, 
    - Groups : 권한 묶음, 벡엔트팀, 프론트엔드팀,,
    - IAM Role : 프로그램/서버에 주는 권한
    - Policies : 무엇을 허용하고 거부할지 규칙

# github action : GitHub 안에서 자동으로 작업해줌(github에 코드를 올리면, 자동으로 테스트 돌리고, 빌드, 서버에 배포, 코드 품질 검사)

# CI/CD 
    - CI : 코드 테스트 자동화
    - CD : 배포 자동화
